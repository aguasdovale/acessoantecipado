00 — Sobre o Projeto

O QUE É O ÁGUAS DO VALE

O projeto Águas do Vale propõe o desenvolvimento de uma rede de estações de monitoramento contínuo da qualidade da água do Rio Paraíba do Sul, implantadas nos municípios de São José dos Campos, Pindamonhangaba e Guaratinguetá. Do ponto de vista do hardware, cada estação é composta por dois microcontroladores Arduino UNO operando como dispositivos escravos, responsáveis pela leitura dos sensores de pH, oxigênio dissolvido, turbidez e condutividade elétrica via protocolo UART, e por um ESP32800L atuando como controlador mestre, integrando ainda um sensor de temperatura DS18B20, módulo RTC DS3231 para registro de data e hora, cartão SD para armazenamento local e display LCD 20×4 para visualização em campo. A transmissão dos dados para a nuvem é realizada via módulo GSM integrado ao ESP32800L, eliminando a dependência de redes WiFi locais e garantindo conectividade contínua mesmo em áreas de infraestrutura limitada. Todo o sistema opera de forma autônoma com painel solar de 30W e bateria de 12V 20Ah.

Do ponto de vista do software, os dados coletados são enviados em tempo real para a plataforma Arduino Cloud, onde são processados, armazenados e disponibilizados por meio de um dashboard web e de um aplicativo mobile, permitindo o acompanhamento simultâneo dos três pontos de monitoramento com atualização contínua dos parâmetros físico-químicos.

01 — Arquitetura do Sistema

VISÃO GERAL DO HARDWARE

Dois Arduinos UNO coletam os dados dos sensores e enviam ao ESP32 via UART. O ESP32 centraliza, grava no SD, exibe no LCD e envia para a nuvem via GSM. Tudo alimentado por painel solar + bateria.

Filtro digital

EMA — Exponential Moving Average

Todos os sensores usam filtro passa-baixa digital com α=0.05, eliminando ruídos do ADC sem atrasar as leituras reais.

  • Amostras pH100 × 2ms
  • Amostras turbidez500 × 1ms
  • Alpha (suavização)0.05
Calibração

Pontos de Calibração em Campo

Calibração com soluções tampão antes de cada período de medição, com valores gravados no SD Card com timestamp.

  • pHTampão 4.0 e 7.0
  • EC1413 µS/cm padrão
  • TurbidezÁgua destilada 0 NTU
Classificação CONAMA

Padrão de Referência

Os dados são comparados aos padrões da Resolução CONAMA 357/2005 para rios de Classe 2.

  • pH6.5 – 8.5
  • OD mínimo≥ 5.0 mg/L
  • Turbidez máx.≤ 100 NTU

Formato JSON — Arduino#1

Protocolo de Comunicação UART

O ESP32 envia o caractere 'R' (request) a cada 5s. O Arduino responde com um JSON compacto terminado em \n.

  • Arduino#1 responde{"ph":7.23,"od":8.5,"odt":25.1}
  • Arduino#2 responde{"ntu":3.2,"ec":248,"ect":24.8}
  • Timeout3 segundos
  • Ciclo de polling5 segundos
Formato CSV — SD Card

Log Local de Segurança

Todos os dados são gravados em agua.csv no cartão SD com timestamp do RTC DS3231, garantindo registro mesmo sem WiFi.

  • Headerdatetime,ph,od,ntu,ec,t°
  • Intervalo5 segundos
  • Formato dataYYYY-MM-DD HH:MM:SS
  • Capacidade>1M leituras/GB

06 — Hardware

COMPONENTES POR ESTAÇÃO

🟢
Master · WiFi · Cloud

ESP32

Centraliza dados, grava SD, exibe LCD, envia para Arduino Cloud. GPIO16/17/18/19 UART, GPIO21/22 I²C, GPIO23 OneWire.

🔵
Escravo #1 · pH + OD

Arduino UNO #1

Lê pH no A0 com 100 amostras/ciclo. Comunica com Shield OD via SoftwareSerial D2/D3. Responde JSON ao ESP32.

🟣
Escravo #2 · Turbidez + EC

Arduino UNO #2

Lê turbidez no A1 com 500 amostras/ciclo. Comunica com Shield EC via SoftwareSerial D2/D3. Responde JSON ao ESP32.

☀️
Energia autônoma

Solar 30W + 12V 20Ah

Painel solar 30W + bateria VRLA 12V 20Ah. Autonomia >4 dias sem sol. Controlador PWM 10A + conversor 12→5V 3A.

🛢️
Câmara de medição

Sumidouro 325L

Rotomold PEBDL preto. Ø70×98cm. Semi-enterrado na margem. Furos como stilling well natural. 15kg vazio.

📦
Proteção IP66

Caixa Hermética

Tampa transparente com prensa-cabos IP68. Aparafusada na tampa do sumidouro com silicone. Eletrônica acima d'água.

💾
Backup local

RTC + SD Card

DS3231 garante timestamp preciso mesmo sem WiFi. SD Card grava CSV com todas as leituras. Backup independente da nuvem.

🖥️
Interface local

LCD 20×4 I²C

3 telas rotativas: pH+OD / Turbidez+EC / Temperatura+Data. Atualização a cada 4 segundos. Endereço I²C 0x27.

07 — Roadmap

PRÓXIMOS PASSOS

1

MONTAGEM E TESTES EM BANCADA

Integrar todos os módulos, testar comunicação UART entre os 3 dispositivos, calibrar sensores com soluções padrão e validar o envio de dados para o Arduino Cloud.

2

PREPARAÇÃO DAS CÂMARAS (SUMIDOUROS)

Adquirir 3 sumidouros Rotomold 325L, instalar barras de suporte internas, prensa-cabos, caixas IP66 nas tampas e testar vedação completa com imersão.

3

INSTALAÇÃO NO CAMPO — SJC (PILOTO)

Instalar o primeiro ponto em São José dos Campos como piloto. Validar dados contra amostras laboratoriais coletadas simultaneamente.

4

EXPANSÃO PARA PINDA E GUARATINGUETÁ

Após validação do piloto, instalar os outros dois pontos. Configurar dashboard comparativo com os 3 pontos simultâneos.
